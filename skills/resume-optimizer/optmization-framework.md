# Resume Optimization Framework

This framework provides a systematic, multi-phase approach to resume optimization based on proven methodology.

## Phase 1: Requirement Analysis

Extract and categorize all requirements from the job posting:

1. **Required qualifications** - Must-have skills, certifications, experience
2. **Preferred qualifications** - Nice-to-have items that strengthen candidacy
3. **Key responsibilities** - Core functions of the role
4. **Technical skills** - Specific tools, frameworks, platforms
5. **Soft skills** - Leadership, communication, collaboration needs

Create a mapping table showing:
- Requirement/Qualification from JD
- Where it's addressed in current resume (if at all)
- Gap analysis (missing, weak, or strong coverage)

## Phase 2: Historical Resume Mining

Search the resume-bullets.md for relevant bullet points:

```
Search queries to run:
- Specific frameworks mentioned in JD (e.g., "NIST 800-53", "SOC2")
- Role-specific keywords (e.g., "GRC", "compliance program", "audit")
- Company names if applying to similar roles
- Technical certifications mentioned
```

Extract strong bullet points and phrasing that can be adapted for this application.

## Phase 3: ATS Optimization

Ensure resume passes Applicant Tracking Systems:

1. **Keyword density** - Include exact phrases from JD (especially in required qualifications)
2. **Standard section headers** - Use common headers (Summary, Professional Experience, Education, Certifications)
3. **Format compatibility** - Avoid tables, text boxes, headers/footers
4. **Acronym handling** - Spell out on first use, then acronym (e.g., "Governance, Risk, and Compliance (GRC)")
5. **Date formatting** - Use consistent MM/YYYY format

## Phase 4: Content Enhancement

Apply Max's writing principles to every bullet:

1. **Lead with impact** - Start with the result or achievement
2. **Active voice** - Use strong action verbs
3. **Quantify everything** - Add metrics, percentages, team sizes, dollar amounts
4. **Be concise** - Remove unnecessary words
5. **Avoid flowery language** - Focus on concrete experience

### Bullet Point Formula

```
[Action Verb] + [What] + [How/Context] + [Quantified Result]

Example:
"Established IT SOX compliance program managing 50+ controls across 15 applications, reducing audit findings by 40% and achieving zero material weaknesses for 3 consecutive years"
```

## Phase 5: Strategic Positioning

1. **Reorder experience** - Lead with most relevant roles/achievements
2. **Tailor job descriptions** - Emphasize aspects that match JD requirements
3. **Section prioritization** - Move certifications up if heavily emphasized in JD. Confirm before deviating from established section structure
4. **Skills section** - Mirror JD language for technical skills

## Quality Checklist

Before finalizing:
- [ ] Every required qualification addressed
- [ ] 80%+ of preferred qualifications covered
- [ ] Key JD phrases appear verbatim (for ATS)
- [ ] All bullets quantified where possible
- [ ] No bullets exceed 2 lines
- [ ] Consistent formatting throughout
- [ ] Recent/relevant experience leads
